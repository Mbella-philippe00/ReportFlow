# User Guide

## Dashboard

The dashboard summarizes weekly report progress, pending workflow queues, recent reports, and workflow KPIs.

## Weekly Reports

1. Create a draft weekly report.
2. Add objectives, activities, achievements, difficulties, next actions, and optional executive summary.
3. Submit the report when ready.
4. Track progress in the timeline.
5. Review manager comments and status badges.

## Workflow States

- **Draft:** Editable by the owner.
- **Submitted:** Awaiting manager review.
- **Under Review:** Manager-approved and waiting for final approval.
- **Approved:** Final, read-only, and decision-ready.
- **Rejected:** Returned with rejection context.

## Documents and Collaboration

Users can upload report attachments, view supported previews, add new versions, inspect metadata, comment in threads, and mention collaborators.

## Notifications

Notifications appear for workflow changes, document collaboration, mentions, and administration-relevant events. Users can mark notifications as read, archive them, restore them, or delete them.

## AI Assistant

AI tools help summarize reports, improve writing, detect action items, assess risk, suggest missing information, and recommend workflow actions. The UI displays the active provider and fallback status when available.

## Analytics

Analytics views provide report trends, employee and department comparisons, workflow metrics, executive KPIs, alerts, and exports where authorized.
